If you're making a sophisticated website with forms and user login, you'll need two things: a web server and an app server. We've already walked through setting up an app server in Launching a Web Server from Scratch.

Websites with advanced features like user login and database interaction can be built using one of many different frameworks, including .NET, Java, Ruby on Rails, and more. One of the most popular frameworks for launching Python websites today is called Django. The purpose of this post is to walk you through deploying a working Django app to a production server. We'll use a tool called Gunicorn (our app server) to run our Django app, which will communicate with NGINX (our web server). Note that Gunicorn will run any kind of WSGI app, which includes Flask in addition to Django.

Note that tools like AWS's Elastic Beanstalk are also available for deploying Django projects, but may cost more and hide some advanced functionality from you. Instead, for this tutorial we'll deploy our project to a web server by hand.

Part 1: Configuring Gunicorn

  1. Inside your Django project, look for a file called wsgi.py—it should have automatically been placed there when you created your Django project. We'll point Gunicorn at this file to run it on your app. Note down the name of the folder this wsgi.py file is in, e.g. my-main-app.
  2. Run sudo apt install gunicorn.
  3. Run the following: {% highlight sh %} sudo su cd location/of/your/Django/project gunicorn --workers 4 --bind 0:8000 my-main-app.wsgi:application {% endhighlight %}
